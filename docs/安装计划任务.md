# 开机补跑：安装说明

脚本 `run_ticktick_daily.ps1` 每天凌晨 1 点后运行，取昨天数据，写日记并推钉钉。同一天内不会重复执行。

## 用 Windows 计划任务设置

1. 按 `Win + R`，输入 `taskschd.msc`，回车打开「任务计划程序」。
2. 右侧点「创建基本任务」。
3. 名称：`TickTick 日记 开机补跑`（或 `TickTick 日记 + 钉钉`），下一步。
4. **触发器（推荐）**：选 **「每天」**，下一步 → 开始时间设为 **凌晨 1:05**，下一步。
5. 操作：选「启动程序」，下一步。
6. 程序或脚本填：`powershell.exe`
7. 添加参数：
   ```text
   -NoProfile -ExecutionPolicy Bypass -File "C:\Users\你的用户名\git\ticktick2md2ding\run_ticktick_daily.ps1"
   ```
   请把路径改成你本机 **run_ticktick_daily.ps1 所在目录的完整路径**（若放在 Obsidian 日记目录则用 `...\gorf_obsidian\日记\run_ticktick_daily.ps1`）。
8. 下一步 → 完成。

### 可选：同时加「用户登录时」或「工作站解锁时」补跑

- **用户登录时**：开机/冷启动后登录会运行（休眠恢复后解锁不会触发）。
- **工作站解锁时**：休眠或锁屏后解锁会运行，适合经常休眠的用户。

任务属性 → 「触发器」→ 新建 → 选「当用户登录时」或「工作站解锁时」→ 选择用户 → 确定。

### 建议设置

- 「常规」→ 勾选「使用最高权限运行」；配置为 **Windows 10/11**。
- 「条件」→ 若电脑常休眠，可取消「只有在计算机使用交流电源时才启动」。
- 若希望凌晨 1 点即使用户未开机也执行：编辑「每天」触发器，勾选 **「唤醒计算机以运行此任务」**（需系统允许唤醒定时器）。

## 手动测试

在 PowerShell 里进入脚本所在目录后执行：

```powershell
cd "C:\Users\你的用户名\git\ticktick2md2ding"
.\run_ticktick_daily.ps1
```

若当前时间早于 1:00，脚本会直接退出；把 `run_ticktick_daily.ps1` 里 `if ($Now.Hour -lt 1)` 暂时改成 `if ($false)` 再运行即可测试。

## 说明

- 同一天内只跑一次：用 `.last_ticktick_run` 记录上次生成的日期。
- 钉钉链接会写入对应日期的日记 `YYYY-MM-DD.md` 的「钉钉工作日志」小节；链接需**复制到浏览器或钉钉**打开。

## 自动运行不生效时

1. **确认触发器**：任务属性 → 触发器 → 应有「每天 1:05」或类似。若只有「用户登录时」，且你 1 点前就登录了，当天不会跑。
2. **电脑 1 点时须开机**：计划任务在关机/休眠时不会执行。可加「用户登录时」或「工作站解锁时」触发器做补跑。
3. **看日志**：脚本同目录下 `.last_ticktick_run.log` 记录最近一次运行结果：`OK` 成功、`SKIP` 跳过、`EXIT`/`ERROR` 失败（末尾有 Python 报错）。
4. **手动跑一遍**：在 PowerShell 中进入脚本目录执行 `.\run_ticktick_daily.ps1`，若有报错会直接显示。
5. **生成的是哪天的文件**：脚本始终处理**昨天**。例如 3 月 9 日凌晨跑 → 生成 `2026-03-08.md`。当天日记要等第二天凌晨才会被自动填充。
6. **计划任务显示 (0x1)**：若 Python 的 WARNING 写 stderr 导致 PowerShell 报错，当前版脚本已做兼容（临时关闭 Stop 模式），更新到最新 `run_ticktick_daily.ps1` 即可。
