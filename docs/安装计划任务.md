# 开机补跑：安装说明

脚本 `run_ticktick_daily.ps1` 每天凌晨 1 点后运行，取昨天数据，写日记并推钉钉。同一天内不会重复执行。

## 用 Windows 计划任务设置

1. 按 `Win + R`，输入 `taskschd.msc`，回车打开「任务计划程序」。
2. 右侧点「创建基本任务」。
3. 名称：`TickTick 日记 开机补跑`，下一步。
4. **触发器（推荐）**：选 **「每天」**，下一步 → 开始时间设为 **凌晨 1:05**，下一步。
5. 操作：选「启动程序」，下一步。
6. 程序或脚本填：`powershell.exe`
7. 添加参数：
   ```text
   -NoProfile -ExecutionPolicy Bypass -File "C:\Users\你的用户名\Documents\gorf_obsidian\日记\run_ticktick_daily.ps1"
   ```
   请把路径改成你本机 `日记` 文件夹下 `run_ticktick_daily.ps1` 的**完整路径**（若在 git 项目则用 `...\git\ticktick2md2ding\run_ticktick_daily.ps1`）。
8. 下一步 → 完成。

### 可选：同时加「用户登录时」补跑

若希望开机/登录时也尝试运行（例如电脑 1 点没开、早上才开机）：  
任务属性 → 「触发器」→ 编辑 → 勾选「已启用」；或新建触发器，选「当用户登录时」。

### 建议设置

- 「常规」→ 勾选「使用最高权限运行」；配置为 **Windows 10/11**。
- 「条件」→ 若电脑常休眠，可取消「只有在计算机使用交流电源时才启动」。

## 手动测试

在 PowerShell 里进入日记目录后执行：

```powershell
cd "C:\Users\你的用户名\Documents\gorf_obsidian\日记"
.\run_ticktick_daily.ps1
```

若当前时间早于 1:00，脚本会直接退出；把 `run_ticktick_daily.ps1` 里 `if ($Now.Hour -lt 1)` 暂时改成 `if ($false)` 再运行即可测试。

## 说明

- 同一天内只跑一次：用 `.last_ticktick_run` 记录上次生成的日期。
- 钉钉链接会写入对应日期的日记 `日记/YYYY-MM-DD.md` 的「钉钉工作日志」小节；链接需**复制到浏览器或钉钉**打开，在 Cursor 里直接点会报错。

## 自动运行不生效时

1. **确认触发器**：任务属性 → 触发器 → 应有「每天 1:05」或类似。若只有「用户登录时」，且你 1 点前就登录了，当天不会跑。
2. **电脑 1 点时须开机**：计划任务在关机/休眠时不会执行。若常关机，可加「用户登录时」触发器做补跑。
3. **看日志**：`日记/.last_ticktick_run.log` 记录最近一次运行结果（OK / EXIT / ERROR）。
4. **手动跑一遍**：在 PowerShell 中执行 `cd "C:\Users\你的用户名\Documents\gorf_obsidian\日记"; .\run_ticktick_daily.ps1`，若有报错会直接显示。
